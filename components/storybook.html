Django Blog Components WCAG 2.1 AA

A collection of reusable, accessible UI components for Django applications

Features

  • Accessibility First: All components include ARIA attributes and follow WCAG 2.1 guidelines
  • Customizable: Easy to customize with CSS variables and Bootstrap classes
  • Responsive: Mobile-first design approach
  • Reusable: Template tags for easy integration
  • Well Documented: Comprehensive documentation and examples

Button Accessible

Buttons are used for actions and navigation. They include proper ARIA labels and support for disabled states.

Preview

Code Example

{% load ui_components %}
{% button "Click Me" "primary" "lg" aria_label="Submit form" %}

Parameters

Parameter Type Default Description
text string - Button text
button_type string primary Button style (primary, secondary, success, danger, warning, info, light, dark, link)
size string md Button size (sm, md, lg)
disabled boolean false Whether the button is disabled
aria_label string - Accessible label for screen readers
icon string - Icon class (e.g., 'fas fa-check')

Modal Accessible

Modals are used to display content in a layer above the app. They include proper focus management and keyboard navigation.

Preview

Parameters

Parameter Type Default Description

title string - Modal title

Form Field Accessible

Preview

Parameter Type Default Description
field form field - Django form field
label string - Field label
help_text string - Help text for the field
error_message string - Error message
